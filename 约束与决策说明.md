# 约束与决策说明

本文记录项目开发中各类客观约束下的技术与功能取舍逻辑。  

## 1.技术栈约束
自身技术栈为Qt + C++，原生Android Java开发经验不足，因此放弃纯原生安卓开发，选用Qt for Android方案，并使用OpenCV图像处理代码，大幅降低跨端适配学习成本，保障任务按时交付。

## 2.时间周期约束
开发周期有限，优先完成题目硬性强制需求：多格式图片兼容、三大画质评分模块、格式差异说明、算法原理与局限性文档、测试样本对比、失准反例分析；舍弃深度学习拓展、批量相册、美化 UI 等非必做功能，集中资源落地核心指标。

## 3.大图性能约束
要求稳定加载 12MP 大图无 OOM，采用分层计算平衡速度与精度：预览用缩略图防卡顿；清晰度基于原图计算保证精度；曝光、对比度使用降采样图运算，常规图片分析耗时控制在 1s 内。  

## 4.算法与附加取舍
受工期与部署限制，全程仅使用传统 OpenCV 算子，并尽可能优化，提高图片判断准确性，不接入深度学习模型，省去模型部署、推理适配工作量。  
仅保留单张图片分析，不做相册管理；UI 仅保留基础功能，无美化设计。针对传统算法固有误判问题，收集失准样本用于局限性分析，客观体现方案短板。

整体所有取舍均围绕跨端兼容、大图稳定、按期交付三大核心目标，在有限条件下完整落地全部核心画质检测需求。
